# 大厂owner养成记-技术规划篇

众所周知，大厂只会做需求是远远不够的，要卷规划，卷文档，卷产出。而晋升管理，时也命也。但owner 是人人都能做到的。

 owner 核心要做好 4 个事情: 技术规划，沟通协作，项目管理，结构化表达。这篇聊聊技术规划。

## 技术规划三板斧，目标，手段，里程碑

### 定目标

大多数同学定目标时，习惯性考虑目前的投入人力，设定的目标一眼望去就可以达成，这怎么行。

要摒除杂念，按照业内最合理的方式来。

目标激进点没事，就怕目标优先级不对。平时开发的痛点，线上的故障，告警，都会影响优先级，当然多和 ld 多聊，给他输入日常开发协作的 context，一般都会给些优先级的建议。

### 找手段

大多数同学看了几篇其他业务的治理经验就开始写，事前事中事后罗列得巨细无遗，这怎么行。得分析现状，找到现状最差的几个点，分析总结一波。针对该痛点定义几个解决手段。

### 定里程碑

大多数人都忽略这一步，一个目标做两三个月。给人感觉就是开头猛如虎，过程没辛苦，结果给惊吓。定好里程碑，分阶段给出结果，一切尽在掌握中。

### 举个万精油的例子，治理稳定性

- 定目标，事故数 x→ 0 事故，告警数 y→z
- 找手段，以事前事中事后时间线遍历。

看事前有些环节是最痛的，是告警召回问题，还是告警发现问题。看事中，是定位问题慢，还是发布慢，还是系统架构问题多，还是操作规范问题。看事后，是不是重复问题多，没总结沉淀。

- 定里程碑，用于拆分季度，半年目标。先堵增量，把线上问题清 0；再全面治理追平部门/公司平均线；最后再常态化保持稳步提升基线

### OKR 示例：

- 年度 KR:  完成稳定性 3 阶段治理，通过手段 1，手段 2，手段 3，事故数 x→ 0 事故，告警数 y→z。
- 拆分 KR: 完成稳定性一阶段治理，通过手段 1 ，手段 2，手段 3，达成事故数 x→ 0 事故。
