# 计算机中一些有意思的抽象

## io = 文件

计算机的 io 设备，比如网卡，磁盘，usb等，这些 io 设备都抽象成字节流，能够打开，关闭，读取，写入，定位，操作系统中用文件表示。对用户屏蔽硬件信息，所有的设备都是文件。

## io +内存 = 虚拟内存

内存是程序运行时的介质，程序运行时有数据，代码，堆栈等信息，程序不运行时可以调度到磁盘外设，操作系统中抽象出虚拟内存概念，对用户屏蔽调度细节，让每个任务看起来是独占完整内存，用户的每个任务按需复用物理内存。

## io +内存+ cpu = 进程

程序运行时，cpu 读取代码段，跳转到函数所在地址，开始执行计算，再将计算结果存到数据段，最后再返回调用处。整个过程涉及到 cpu，内存，io，操作系统抽象出进程的概念，屏蔽复杂的计算机执行过程。多进程之间按需复用硬件资源。

## io +内存+ CPU + os = 虚拟机

操作系统对用户提供文件，虚拟内存，进程概念，对用户屏蔽了硬件细节，给用户提供多任务多用户的环境。同一套硬件上支持多个操作系统，这个叫虚拟机，操作系统间按需复用硬件资源。

## io +内存+ CPU + os + 应用程序 = 容器

应用程序在操作系统上，仍然需要做一些程序配置，部署等操作，将应用程序运行环境和配置进一步封装，解决应用程序运行环境漂移问题，可以让应用程序开箱即用，便于快速扩容调度，这个就叫容器，docker 几乎是容器的代名词了。
