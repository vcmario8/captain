# 初识 AI 连接器——MCP

举个例子，你的红米手机内存不够了，需要外接硬盘U 盘，你需要买个 usb 转换口， type-C 口转 USB-A 母口，就能链接外部 U 盘了。这时你又想听歌，手上有一根 3.5mm 口的耳机，你得把 U 盘拔下来，同时再找个 type-C 转 3.5mm 的转换口，就能链接耳机听歌了……

如果换到了苹果的 lightning 接口，还得重新买这些转换器……

2025 年，苹果终于支持 type-C 口了，市场上较新的手机和外设，都支持了 type-C 到 type-C 了，终于只要一根线就可以链接所有外设了，这种感觉挺爽。

## Q：什么是 MCP？
A：就是上面提到的 type-C 到 type-C 转换器，mcp 规定了，llm 和外部资源交互，需要用到的协议。比如你问大模型，北京的实时天气是什么？llm 就会去天气服务查询，用到的协议就是 mcp。概念比较无聊，Model Context Protocol，模型上下文协议。它的作用是规范 llm 如何与外部资源交互。

## Q：为什么需要 MCP？
A：如果没有 mcp，你就得买各种转换器；对应到智能体 agent，你就得写不同的 prompt；prompt 要细致到具体怎么调用天气服务；天气服务接口变一下，agent 就不能用，得重新适配接口。有了 mcp 后，prompt 大幅精简了，让 agent 更聚焦到思考和决策，执行层面细节就让 mcp 去托管了。

没有 mcp 的 prompt：
查天气需用固定格式调用工具：【{"tool":"weather_query","params":{"city":"城市","date":"YYYY-MM-DD"}}】。缺参数要追问，返回结果转自然语言，格式错则回复"查询失败"。

有 mcp 的 prompt：
查天气可通过MCP调用工具，缺信息用MCP补问，结果转自然语言，失败按MCP错误码处理。
