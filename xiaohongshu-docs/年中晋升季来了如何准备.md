# 年中晋升季来了，如何准备？

## 1. 了解能力雷达（以鹅厂和宇宙的职级为例）

2-1（8-9 级）：靠谱的工程师，可以独立完成中等规模复杂的需求，强调的是独立干活。
2-2（10）：需求考虑、方案设计、代码设计、上线、运营等各个流程，都能做到胸有成竹，都能在计划之内完成；强调的是完备性和计划性、一切尽在掌握中。
3-1（11）：业界先进性，方案的对比具备业界思维，并且灵活结合业务特点做出了权衡，取得了超业界的业务效果。

## 2. 准备思路

从 3 个思路去准备文档，业务、技术、规划。

### 业务部分

篇幅不要太大，更多展示的是业务特点和发展阶段，如果有技术影响业务的部分，可以强调下。

### 技术部分

需要准备 2-3 个项目，不管目标职级多少，都建议去准备业界调研，把自己的方案和业界对比，会让人眼前一亮。每个项目要结构化呈现出来：背景、冲突（难点）、业内解决方案、你的方案概述（结合前面业务特点来权衡）、关键模块的设计（2-3 个）、效果、未来展望；建议前期用思维导图来构建这个框架、然后完善文档。

### 规划部分

可以单独讲讲思路；也可以融入到前面的技术和业务部分；这部分要思考下：规划为什么要跨度这么长？为什么这个 Q要做这个事情，而不是下个 Q？

## 3. 表达呈现

根据思路完善文档后，把呈现的思路整理下，一般答辩下场 60min，主动呈现 40min；20minQA；把自己要展示的重点在 40min 讲完，建议准备一份演讲稿；演讲稿演练多几次，做到脱稿后依旧能思路清晰；并且可以应对随时打断提问。

## 4. QA 准备

这个不是为了押题，而是提前适应问题，以及掌握应对问题的思路。比如问题可能是：为什么你采用这个方案？xx 业务的方案你们考虑过吗？应对思路可以统一下：肯定问题、展开对方方案、阐述自己的方案（权衡点）、再次肯定问题。

### 给个示例

如果你会了：这个问题比较关键，刚好命中了方案关键点；对方的方案我们调研过，xxxx；结合我们 xxx 特点，我们借鉴了 xxx 部分，我们的方案是 xxx；评委看看是否还有其他疑问。
如果你真的不会：这个问题比较关键，刚好命中了方案关键点；评委提的方案，我们没了解过，不过我们调研了 xxx 方案，这类问题的解法都比较类似，xxxx；结合我们 xxx 特点，我们借鉴了 xxx 部分，我们的方案是 xxx；评委看看是否还有其他疑问。
